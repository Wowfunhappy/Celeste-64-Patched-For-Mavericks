Walking a Chain

Algorithm: 1. For each segment with seg_info_offset[i] != 0, read dyld_chained_starts_in_segment. 2. For each page where page_start[j] != DYLD_CHAINED_PTR_START_NONE: - Compute the address of the first fixup: segment_offset + (j * page_size) + page_start[j] - Read the 64-bit (or 32-bit) value at that file offset. - Check the bind bit to determine rebase vs bind. - Extract next. If next == 0, chain ends. Otherwise, advance by next * stride bytes to the next fixup.

Rebase Opcodes

The rebase info is a bytecode stream. Each byte has a 4-bit opcode in the high nibble and a 4-bit immediate in the low nibble.

#define REBASE_TYPE_POINTER         1
#define REBASE_TYPE_TEXT_ABSOLUTE32  2
#define REBASE_TYPE_TEXT_PCREL32     3

// Opcodes (high 4 bits)
#define REBASE_OPCODE_MASK                             0xF0
#define REBASE_IMMEDIATE_MASK                          0x0F

#define REBASE_OPCODE_DONE                             0x00
#define REBASE_OPCODE_SET_TYPE_IMM                     0x10  // type = imm
#define REBASE_OPCODE_SET_SEGMENT_AND_OFFSET_ULEB      0x20  // seg_index = imm, offset = ULEB128
#define REBASE_OPCODE_ADD_ADDR_ULEB                    0x30  // offset += ULEB128
#define REBASE_OPCODE_ADD_ADDR_IMM_SCALED              0x40  // offset += imm * pointer_size
#define REBASE_OPCODE_DO_REBASE_IMM_TIMES              0x50  // do rebase imm times, each advancing by pointer_size
#define REBASE_OPCODE_DO_REBASE_ULEB_TIMES             0x60  // do rebase ULEB128 times
#define REBASE_OPCODE_DO_REBASE_ADD_ADDR_ULEB          0x70  // do 1 rebase, then offset += ULEB128 + pointer_size
#define REBASE_OPCODE_DO_REBASE_ULEB_TIMES_SKIPPING_ULEB 0x80 // rebase ULEB times, each skip ULEB bytes + ptr_size

ULEB128 encoding: Standard unsigned LEB128. Each byte contributes 7 bits; high bit means "more bytes follow."

Typical rebase stream you would emit:

SET_SEGMENT_AND_OFFSET_ULEB(seg, offset)   // 0x20 | seg, then ULEB128(offset)

... repeat ...
DONE                                       // 0x00

Weak Bind Info

Same opcode format as regular bind. Used for symbols with BIND_SYMBOL_FLAGS_WEAK_IMPORT or weak definitions. Goes in weak_bind_off/weak_bind_size.

Lazy Bind Info

Also same opcode format, but each lazy symbol's opcodes are self-contained (they start from scratch with SET_DYLIB, SET_SYMBOL, etc. and end with DONE). The __la_symbol_ptr section's entries contain offsets into the lazy bind area. Each stub jumps through the pointer, and the initial pointer value causes dyld_stub_binder to be called, which uses the offset to find the right lazy bind record.

For your converter, if the chained fixups dylib doesn't use lazy binding (many modern dylibs don't -- they use non-lazy binding for everything), you can put all binds in the regular bind section and leave lazy_bind empty.

The trie data format is identical whether it comes from LC_DYLD_EXPORTS_TRIE or from the export_off/export_size fields in LC_DYLD_INFO_ONLY. The only difference is where the offset/size is specified.

Trie Binary Format

The trie is a byte-serialized prefix tree. Each node has:

node:
    ULEB128 terminal_size     // 0 if not a terminal (no exported symbol ends here)
    if terminal_size > 0:
        ULEB128 flags         // export flags
        if flags & EXPORT_SYMBOL_FLAGS_REEXPORT:
            ULEB128 ordinal   // dylib ordinal for re-export
            char[] name       // null-terminated imported name (or empty for same name)
        else if flags & EXPORT_SYMBOL_FLAGS_STUB_AND_RESOLVER:
            ULEB128 stub_offset    // offset of stub from mach_header
            ULEB128 resolver_offset
        else:
            ULEB128 symbol_offset  // offset from mach_header for regular exports
    uint8_t child_count       // number of child edges
    for each child:
        char[] edge_label     // null-terminated edge string (label characters)
        ULEB128 child_offset  // offset from start of trie data to child node

Export Flags

#define EXPORT_SYMBOL_FLAGS_KIND_MASK           0x03
#define EXPORT_SYMBOL_FLAGS_KIND_REGULAR        0x00
#define EXPORT_SYMBOL_FLAGS_KIND_THREAD_LOCAL   0x01

Conversion Strategy Summary

Here is the high-level algorithm for your converter:

Step 1: Parse the chained fixups

  • Read the linkedit_data_command for LC_DYLD_CHAINED_FIXUPS.
  • Parse dyld_chained_fixups_header, then dyld_chained_starts_in_image.
  • For each segment, walk each page's chain.
  • For each fixup, check the bind bit:
  • bind=0 (rebase): Record (segment_index, offset_in_segment, target_vmaddr). For PTR_64_OFFSET, the target is mach_header_vmaddr + target_field.
  • bind=1 (bind): Look up the import by ordinal. Record (segment_index, offset_in_segment, lib_ordinal, symbol_name, addend).

Step 2: Generate rebase opcodes

  • Sort rebases by (segment_index, offset).
  • Emit SET_TYPE_IMM(POINTER).
  • For each rebase, emit SET_SEGMENT_AND_OFFSET_ULEB (if segment changed), then ADD_ADDR_ULEB or DO_REBASE_IMM_TIMES as appropriate.
  • Emit DONE.

Step 3: Generate bind opcodes

  • Group binds by (lib_ordinal, symbol_name), sorted by (segment_index, offset).
  • For each group, emit the dylib ordinal, symbol name, type, and then for each location emit segment/offset + DO_BIND.
  • Emit DONE.

Step 4: Fix the actual pointer values in DATA pages

This is critical. With chained fixups, the pointer slots in __DATA/__DATA_CONST contain the encoded chain entries (with next pointers, bind bits, etc.). With traditional rebase/bind: - Rebase locations: The pointer value should be the actual target virtual address (the runtime value before ASLR slide is applied). dyld will add the slide. - Bind locations: The pointer value should be 0 (dyld will fill in the resolved symbol address at load time).

Step 5: Rewrite load commands

  • Replace LC_DYLD_CHAINED_FIXUPS with LC_DYLD_INFO_ONLY.
  • Remove LC_DYLD_EXPORTS_TRIE (fold its data into the export_off/export_size of the new command).
  • Adjust __LINKEDIT segment size/offsets to accommodate the new rebase/bind/export data.
  • Update the sizeofcmds field in the mach_header.

Step 6: Patch __LINKEDIT

  • You will likely need to append the new rebase, bind, and export data to __LINKEDIT (or rewrite it entirely).
  • Update the __LINKEDIT segment's filesize/vmsize and the LC_SYMTAB/LC_DYSYMTAB offsets if they shift.
